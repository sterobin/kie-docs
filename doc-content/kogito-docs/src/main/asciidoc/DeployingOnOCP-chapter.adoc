= Deploying {PRODUCT} services on {OPENSHIFT}
:context: kogito-deploying-on-openshift

// Purpose statement for the assembly
As a developer of business processes and rules, you can deploy {PRODUCT} services on {OPENSHIFT} for cloud implementation. The {PRODUCT} Operator automates many of the deployment steps for you or guides you through the deployment process. You can use the {PRODUCT} command-line interface (CLI) to interact with the {PRODUCT} Operator for deployment tasks.

.Prerequisites
////
// @comment: Removed for now but retaining for use in next docs. (Stetson, 18 Feb 2020)
* One of the following IDEs is installed:
** VSCode (recommended) with the {PRODUCT} https://github.com/kiegroup/kogito-tooling/releases[VSCode Extension] installed for BPMN 2.0 and DMN modeling support
**  Eclipse with BPMN2 Modeler plugin with jBPM Runtime Extension for BPMN 2.0 modeling support
////
* {OPENSHIFT} 4.2 or later is installed.
* The OpenShift project for the deployment is created.
